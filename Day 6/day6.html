Day6 — CSS Combinators

Simple examples of the four main CSS combinators: descendant (space), child (>), adjacent sibling (+) and general sibling (~).

1) Descendant combinator (space)
HTML
Any .item that is inside .descendant (at any depth) will match the selector .descendant .item. 
<div class="descendant">
  <div class="item">Matched (child)</div>
  <div>
    <div class="item">Matched (nested)</div>
  </div>
</div>
Rendered
Matched (child)
Matched (nested)
2) Child combinator (>)
HTML
Only direct children of .parent with class .child are matched by .parent > .child. 
<div class="parent">
  <div class="child">Matched (direct child)</div>
  <div>
    <div class="child">Not matched (grandchild)</div>
  </div>
</div>
Rendered
Matched (direct child)
Not matched (grandchild)
3) Adjacent sibling combinator (+)
HTML
The selector h3 + p matches a <p> only if it immediately follows an <h3>. 
<div class="adjacent">
  <h3>Title A</h3>
  <p>Matched (immediately after) </p>

  <h3>Title B</h3>
  <div>Some other element</div>
  <p>Not matched (not immediate)</p>
</div>
Rendered

Title A

Matched (immediately after)

Title B

Some other element

Not matched (not immediate)

4) General sibling combinator (~)
HTML
The selector h3 ~ p matches any <p> that appears after an <h3> as a sibling (not necessarily immediate). 
<div class="general">
  <h3>Heading</h3>
  <p>Matched (after) </p>
  <div>Other</div>
  <p>Also matched (after, non-immediate)</p>
</div>
Rendered

Heading

Matched (after)

Other

Also matched (after, non-immediate)

Tip: Try changing the HTML structure to see which elements get styled — that helps make combinators intuitive.
